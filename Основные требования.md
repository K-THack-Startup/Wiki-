

## Требования к системе

### 2.1 Технические требования
*   **Backend:** Golang v1.25+, Redis, Redis Queue
*   **Инфраструктура:** Docker, Docker Compose, Nginx, GitLab CI/CD, (Prometheus, Grafana ??), VPS
*   **Хранилище:** PostgreSQL v17+, MinIO (локально развернутое файловое хранилище)
*   **Frontend:** React 18+

### 2.2 Нефункциональные требования

#### Безопасность
*   Шифрование паролей с помощью bcrypt (12+)
*   JWT токены (access/refresh) для сессий с автоматическим продлением, http only, secure куки, если в куки будет храниться
*   Интеграция с LDAP колледжа для автоматической верификации студентов
*   Логи всех авторизаций + IP
*   Rate limiting, макс 100 запросов с одного IP
*   HTTPS

#### Производительность
*   Время загрузки страницы < 2с
*   Обработка запросов < 300мс при нагрузке
*   Поддержка 100 пользователей
*   Фотки конвертировать в webp, сжимать дополнительно еще на 10-20%
*   Добавление пагинации и индексов в БД

#### Совместимость
*   Совместимость платформы с Chrome, Firefox, Safari
*   Адаптивный дизайн для расширений от 320px до 2кpx
*   Правильная работа на мобильных устройствах

#### Сопровождаемость
*   Код должен следовать стандартам как Go, так и React
*   API должно быть задокументировано по OpenAPI
*   Логирование всех действий с разделением по уровням (debug, info, warn, error)
*   Мониторинг метрик по процессору, запросам, памяти
*   Покрытие тестами 30-40% ??

### 2.3 Функциональные требования

#### Аутентификация и авторизация
*   Регистрация внешних пользователей, не с колледжа, с помощью телефона/email
*   Возможность зайти как студент КЦТ (дать возможность войти с помощью своего uid и пароля)
*   Восстановление пароля через email
*   Разделение ролей: участник, организатор, администратор, партнёр

#### Управление пользователями
*   **Создание и редактирование профиля:**
    *   **Обязательные поля:**
        *   Телефон
        *   Дата рождения
        *   Электронная почта
        *   Имя пользователя в Telegram
    *   **Необязательные поля:**
        *   Ключевые навыки
        *   Роль в команде
        *   Пол
        *   Команда для участия
        *   Аватар
        *   О себе
        *   Ссылки на GitHub, портфолио, резюме
        *   (Образование?)
*   Загрузка аватара (макс 5 МБ, форматы: jpg, png, webp)
*   Указание навыков с уровнем владения (начинающий/средний/продвинутый и т.д.)

#### Проекты
*   Создание карточки проекта с обязательными полями (название, описание, стек)
*   Загрузка скриншотов (до 10 шт, макс 5 МБ каждый)
*   Привязка участников команды к проекту
*   Фильтрация по технологиям, дате, хакатону
*   Полнотекстовый поиск по названию и описанию
*   Счётчик просмотров проекта
*   Возможность оставить реакцию (лайк)
*   Подтягивание репозиториев с GitBucket, если проект там лежит, который в рамках внутреннего хакатона например ??

#### Команды
*   Создание команды с указанием капитана
*   Приглашение участников (отправка запроса)
*   Возможность пригласить в уже существующую команду по коду ???
*   Принятие/отклонение приглашений
*   Указание текущего проекта команды
*   Поиск команд по тегам
*   Статус команды (активна, ищет участников, (удаленные команды никак не регаются))

#### Расписание и события
*   Создание события с типом (воркшоп, хакатон, тренировка, митап)
*   Регистрация на событие (если есть ограничение мест)
*   Уведомление за 24 часа и за 1 час до события на почту или в (Telegram ??)
*   Отмена регистрации ??

#### Заявки партнёров
*   Форма подачи заявки с валидацией полей
*   Автоматическое уведомление организаторам в Telegram
*   Смена статуса заявки (новая, рассмотрение, принята, выполнена, отклонена)
*   Комментарии по заявке

#### Админ-панель
*   Модерация новых пользователей (верификация студентов) ??
*   CRUD операции для всех сущностей (проекты, команды, события)
*   Управление ролями пользователей
*   Просмотр статистики (новые пользователи, активность)
*   Логи действий пользователей с фильтрацией
