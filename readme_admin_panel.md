# Админ-панель для управления хакатонами





## Функционал админ-панели

Админ-панель предоставляет организаторам полный контроль над событиями, пользователями и командами.





### Управление событиями

Организатор может:

* **Добавлять события**

  * Заполняет поля:

    * Тип события (хакатон, митап, воркшоп)
    * Название
    * Описание
    * Карточка события
    * Технологический стек
    * Дата проведения
    * Место проведения
    * Максимальное количество участников

  * Нажимает "Добавить событие"
  * Событие сохраняется в базу данных

* **Редактировать события**

  * Видит список событий
  * Выбирает событие → нажимает "Редактировать"
  * Изменяет нужные поля
  * Сохраняет изменения

* **Удалять события**

  * Видит список событий
  * Выбирает событие → нажимает "Удалить"
  * Подтверждает удаление
  * Событие удаляется из базы данных





### Управление пользователями

Организатор может:

* **Просматривать профиль пользователя**

  * Личные данные (имя, email, телефон)
  * Технологический стек
  * Социальные сети
  * Активность на платформе

* **Редактировать профиль пользователя**

  * Заходит в профиль пользователя
  * Нажимает "Редактировать профиль"
  * Изменяет данные
  * Сохраняет изменения
  * Профиль обновлен

* **Удалять пользователя**

  * Заходит в профиль пользователя
  * Нажимает "Удалить профиль"
  * Подтверждает удаление
  * Пользователь удаляется из базы данных





### Управление командами

Организатор может:

* **Просматривать профиль команды**

  * Название команды
  * Описание
  * Название проекта
  * Описание проекта
  * Участники команды
  * Team Lead

* **Редактировать команду**

  * Заходит в профиль команды
  * Нажимает "Редактировать команду"
  * Изменяет данные
  * Сохраняет изменения
  * Профиль команды обновлен

* **Удалять команду**

  * Заходит в профиль команды
  * Нажимает "Удалить команду"
  * Подтверждает удаление
  * Команда удаляется из базы данных





## Таблицы БД

### Таблица: events (события)

|Поле|Тип|Описание|
|-|-|-|
|id|BIGSERIAL|Первичный ключ (автоинкремент)|
|event\_type|VARCHAR(50)|Тип события (HACKATHON, MEETUP, WORKSHOP)|
|title|VARCHAR(255)|Название события|
|description|TEXT|Описание события|
|tech\_stack|TEXT|Технологический стек|
|event\_card|TEXT|Карточка события|
|start\_date|TIMESTAMP|Дата начала|
|end\_date|TIMESTAMP|Дата окончания|
|location|VARCHAR(255)|Место проведения|
|max\_participants|INTEGER|Максимальное количество участников|
|created\_at|TIMESTAMP|Дата создания записи|
|updated\_at|TIMESTAMP|Дата обновления записи|

### Таблица: users (пользователи)

|Поле|Тип|Описание|
|-|-|-|
|id|VARCHAR(36)|Первичный ключ (UUID)|
|email|VARCHAR(255)|Email пользователя (уникальный)|
|first\_name|VARCHAR(100)|Имя|
|last\_name|VARCHAR(100)|Фамилия|
|phone|VARCHAR(20)|Телефон|
|city|VARCHAR(100)|Город|
|profile\_image|TEXT|Ссылка на фото профиля|
|bio|TEXT|О себе|
|tech\_stack|TEXT|Технологический стек|
|github|VARCHAR(255)|Ссылка на GitHub|
|linkedin|VARCHAR(255)|Ссылка на LinkedIn|
|is\_active|BOOLEAN|Активен ли пользователь|
|created\_at|TIMESTAMP|Дата регистрации|
|updated\_at|TIMESTAMP|Дата обновления профиля|

### Таблица: teams (команды)

|Поле|Тип|Описание|
|-|-|-|
|id|VARCHAR(36)|Первичный ключ (UUID)|
|name|VARCHAR(255)|Название команды|
|description|TEXT|Описание команды|
|project\_name|VARCHAR(255)|Название проекта|
|project\_description|TEXT|Описание проекта|
|team\_lead\_id|VARCHAR(36)|ID тимлида (FK к users)|
|is\_active|BOOLEAN|Активна ли команда|
|created\_at|TIMESTAMP|Дата создания|
|updated\_at|TIMESTAMP|Дата обновления|

### Таблица: team\_members (участники команд)

|Поле|Тип|Описание|
|-|-|-|
|team\_id|VARCHAR(36)|ID команды (FK к teams)|
|user\_id|VARCHAR(36)|ID пользователя (FK к users)|
