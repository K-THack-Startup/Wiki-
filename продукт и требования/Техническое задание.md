1. # 1. **КЦТHack** 

      **Тип проекта**: Практико-Ориентированный. 

      **Цель проекта:** 

      1. Создать самовоспроизводящуюся экосистему практического образования в IT, которая готовит конкурентоспособных специалистов через реальные проекты и сообщество.

      2. Стать главной точкой входа в IT для студентов Колледжа Цифровых Технологий и поставщиком талантов для региональных IT-компаний.

      3. Создать рабочее сообщество на 100+ человек к концу учебного года.

      Метрики: 

      - 100+ участников в Телеграмм-канале. 

      - 5+ собранных команд. 

      4. Подготовить 5+ команд для участия во внешних хакатонах и добиться 2+ призовых мест. 

      5. Обеспечить участникам сообщества стажировки и трудоустройства в IT компании.

      6. Построить самоуправляемую систему, чтобы она могла действовать и после выпуска основателей.

      Метрики:

      - 3+ числа организаторов из числа участников. 

      - 80% работы автоматизировано через систему. 

      - Наличие документации и регламентов. 

      **Результат проекта:** 

      - регулярные тренировки для программистов, на базе колледжа, по 3-4 часа, с заранее продуманным планом и продуманными задачами. 

      - Активное участие в хакатонах. 

      - Созданное сообщество:  

      1. сайт

      2. сформированные  команды по 3-5 человек, которые прошли какие либо ивенты колледжа и готовы к участию в хакатонах из вне. 

      3. телеграмм канал для контента с нашими участниками из колледжа с большим количеством подписчиков как колледжа так и наших фанатов 

      4. Составленный план по Маркетингу, которому придерживается вся команда. То Есть активно ведутся все соцсети (вк, тг, тик ток, хабр и тд) 

      5. Телеграмм канал в котором мы будем публиковать расписание тренировочного процесса и так далее. 

      6. Портфолио с рабочими проектами. 

      1) **Введение.** 

      **1.1 Назначение проекта:** 

      Создание централизованной веб-платформы для:

      - Привлечение новых участников в сообщество колледжа. 

      - Автоматизация управления проектами и командами. 

      - Организация взаимодействия с IT-компаниями и партнерами. 

      - Систематизация портфолио проектов студентов. 

      **1.2 Целевая аудитория:**

      - Студенты: разработчики, дизайнеры, сисадмины, ПМ (любого уровня подготовки). 

      - Партнеры: представители IT-компаний, потенциальные работодатели. 

      - Организаторы: ядро команды клуба. 

      - Администрация колледжа: преподаватели. 

      2. **Требования к системе.**

   ### **2.1 Технические требования**

   # 1. **Backend:** Golang v1.25+, Redis_,_ Redis Queue

      **Инфраструктура:** Docker, Docker Compose, Nginx, GitLab CI/CD, _(Prometheus, Grafana ??),_ VPS

      **Хранилище:** PostgreSQL v17+, MinIO (локально развернутое файловое хранилище)

      **Frontend:** React 18+

   ### **2.2 Нефункциональные требования**

   # 1. **Безопасность**

      1. Шифрование паролей с помощью **bcrypt (12+)**

      2. JWT токены _(access/refresh)_ для сессий с автоматическим продлением, http only, secure куки, если в куки будет храниться

      3. Интеграция с LDAP колледжа для автоматической верификации студентов

      4. Логи всех авторизаций + ip

      5. Rate limiting, макс 100 запросов с одного ip

      6. HTTPS

      **Производительность**

      1. Время загрузки страницы < 2с

      2. Обработка запросов < 300мс при нагрузке

      3. Поддержка 100 пользователей

      4. Фотки конвертировать в webp, сжимать допом еще на 10-20%

      5. Добавление пагинации и индексов в БД

      **Совместимость**

      1. Совместимость платформы с Chrome, Firefox, Safari

      2. Адаптивный дизайн для расширений от 320px до 2кpx

      3. Правильная работа на мобилках

      **Сопровождаемость**

      1. Код должен следовать стандартам как Go, так и React

      2. API должно быть задокументировано по OpenAPI

      3. Логирование всех действий с разделением по уровням (debug, info, warn, error)

      4. Мониторинг метрик по процессору, запросам, памяти

      5. _Покрытие тестами 30-40% ??_

   ### **2.3 Функциональные требования**

   # 1. **Аутентификация и авторизация**

      1. Регистрация внешних пользователей, не с колледжа, с помощью телефона/email

      2. Возможность зайти как студент КЦТ _(дать возможность войти с помощью своего uid и пароля)_

      3. Восстановление пароля через email

      4. Разделение ролей: участник, организатор, администратор, партнёр

      **Управление пользователями**

      1. Создание и редактирование профиля:

         1. **Обязательные поля для заполнения:**

            1. Телефон

            2. Дата рождения

            3. Электронная почта

            4. Имя пользователя в Telegram

         2. **Необязательные поля для заполнения:**

            1. Ключевые навыки

            2. Роль в команде

            3. Пол

            4. Команда для участия

            5. Аватар

            6. О себе

            7. Ссылки на GitHub, портфолио, резюме

            8. _(Образование?)_

      2. Загрузка аватара (макс 5 МБ, форматы: jpg, png, webp)

      3. Указание навыков с уровнем владения (начинающий/средний/продвинутый и т.д)

      **Проекты**

      1. Создание карточки проекта с обязательными полями (название, описание, стек)

      2. Загрузка скриншотов (до 10 шт, макс 5 МБ каждый)

      3. Привязка участников команды к проекту

      4. Фильтрация по технологиям, дате, хакатону

      5. Полнотекстовый поиск по названию и описанию

      6. Счётчик просмотров проекта

      7. Возможность оставить реакцию (лайкккк)

      8. _Подтягивание репозиториев с GitBucket, если проект там лежит, который в рамках внутреннего хакатона например ??_

      **Команды**

      1. Создание команды с указанием капитана

      2. Приглашение участников (отправка запроса)

      3. _Возможность пригласить в уже существующую команду по коду ???_

      4. Принятие/отклонение приглашений

      5. Указание текущего проекта команды

      6. Поиск команд по тегам

      7. Статус команды (активна, ищет участников, _(удаленные команды никак не регаются)_)

      **Расписание и события**

      1. Создание события с типом (воркшоп, хакатон, тренировка, митап)

      2. Регистрация на событие (если есть ограничение мест)

      3. Уведомление за 24 часа и за 1 час до события на почту или в (_Telegram ??)_

      4. _Отмена регистрации ??_

      **Заявки партнёров**

      1. Форма подачи заявки с валидацией полей

      2. Автоматическое уведомление организаторам в Telegram

      3. Смена статуса заявки (новая, рассмотрение, принята, выполнена, отклонена)

      4. Комментарии по заявке 

      **Админ-панель**

      1. _Модерация новых пользователей (верификация студентов) ??_

      2. CRUD операции для всех сущностей (проекты, команды, события)

      3. Управление ролями пользователей

      4. Просмотр статистики (новые пользователи, активность)

      5. Логи действий пользователей с фильтрацией

      3) **Интерфейс и дизайн.** 

      3.1 Принципы дизайна:

      Стиль: минимализм, тематика IT.

      Цветовая схема: ярко фиолетовый + черно-темные оттенки. 

      Шрифты: система шрифтов колледжа. 

      3.2 Основные страницы (Макеты):

      - главная страница (лендинг)

      - каталог проектов с фильтрами 

      - страница с описанием “о нас”. 

      - Профили команд и участников.

      - Форма заявки партнера

      - Админ панел.
