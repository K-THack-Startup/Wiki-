## Декомпозиция

### Дизайн

**Спринт 1: Подготовка к началу дизайна.**
*   Понимание целей проекта.
*   Выявление целевой аудитории проекта.
*   Изучение возможных плюсов и минусов конкурентов.
*   Составление User Flow проекта.
*   Сбор референсов.

**Спринт 2: Поиск команд. (2-3 неделя)**
*   Мудборд.
*   Подобранные цвета (с учетом нашей ЦА).
*   Подобранные шрифты.
*   Составленный прототип.

**Спринт 3: Страницы MVP (4-5 неделя)**
*   Главная страница (+ блок о нас + руководители + достижения).
*   Страница "Участники + команды".
*   Расписание событий.
*   ЛК студентов/партнеров.
*   Админ-панель для администратора.

**Спринт 4: Страницы для второго релиза (6-7 неделя)**
*   Взаимодействие с партнером.
*   Страница "Проекты".

**Спринт 5: Страницы для третьего релиза (8 неделя)**
*   Страница с хакатонами + управление хакатонами.

## Планируемые результаты по этапам

**По завершению первого месяца (4 недели)**
*   Рабочая система регистрации и аутентификации.
*   Личные кабинеты пользователей с заполнением профилей.
*   Базовая структура данных.
*   Готовность ~40% функционала MVP.

**По завершению 2 месяца (7 недель)**
*   Полнофункциональный поиск пользователей по фильтрам.
*   Система отправки/принятия приглашений в команды.
*   Механизм создания виртуальных команд.
*   Базовая чат-система для команд.
*   Готовность: ~80% функционала MVP.

**К финальному запуску: (9 недель)**
*   Работающая админ-панель.
*   Расписание событий, возможность добавления события в расписание.
*   Платформа развернута на продакшн-сервере.
*   Первичный контент.
*   MVP готов к запуску.
