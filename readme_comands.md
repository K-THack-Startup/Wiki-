# Система управления командами для хакатонов

## Общая информация

Система предназначена для управления командами участников хакатонов. Реализован функционал создания команд, обработки заявок на вступление, системы уведомлений и регистрации на хакатоны.

## Основные возможности

### Для создателя команды:
- Создание команды
- Получение уведомлений о новых заявках на вступление
- Просмотр списка ожидающих заявок
- Одобрение или отклонение заявок
- Регистрация команды на хакатоны

### Для участников:
- Просмотр доступных команд
- Отправка запросов на вступление в команду
- Получение уведомлений о результатах рассмотрения заявки
- Просмотр своих команд и заявок

## Правила работы

1. **Создатель команды** - пользователь, создавший команду. Только он может:
   - Просматривать входящие заявки
   - Одобрять/отклонять заявки
   - Регистрировать команду на хакатоны

2. **Заявки на вступление**:
   - Любой пользователь может подать заявку в любую команду
   - Нельзя подать заявку, если уже являешься участником команды
   - Нельзя подать повторную заявку, пока предыдущая находится на рассмотрении
   - После одобрения пользователь автоматически становится участником команды

3. **Регистрация на хакатоны**:
   - Только создатель команды может зарегистрировать команду на хакатон
   - Команда может быть зарегистрирована на несколько хакатонов

## Таблицы БД

### Таблица: teams (команды)

| Поле | Тип | Описание |
|------|-----|----------|
| id | BIGINT | Первичный ключ (автоинкремент) |
| name | VARCHAR | Название команды |
| description | VARCHAR | Описание команды |
| creator_id | VARCHAR | ID создателя команды |
| creator_name | VARCHAR | Имя создателя команды |
| created_at | TIMESTAMP | Дата создания |

### Таблица: team_members (участники команды)

| Поле | Тип | Описание |
|------|-----|----------|
| id | BIGINT | Первичный ключ (автоинкремент) |
| team_id | BIGINT | ID команды (внешний ключ) |
| user_id | VARCHAR | ID пользователя |
| user_name | VARCHAR | Имя пользователя |
| joined_at | TIMESTAMP | Дата вступления |
| is_creator | BOOLEAN | Флаг создателя команды |

### Таблица: join_requests (заявки на вступление)

| Поле | Тип | Описание |
|------|-----|----------|
| id | BIGINT | Первичный ключ (автоинкремент) |
| team_id | BIGINT | ID команды (внешний ключ) |
| user_id | VARCHAR | ID пользователя |
| user_name | VARCHAR | Имя пользователя |
| message | VARCHAR | Сопроводительное сообщение |
| status | ENUM | Статус (PENDING, APPROVED, REJECTED) |
| created_at | TIMESTAMP | Дата создания заявки |
| processed_at | TIMESTAMP | Дата обработки |

### Таблица: team_hackathons (регистрации команд на хакатоны)

| Поле | Тип | Описание |
|------|-----|----------|
| id | BIGINT | Первичный ключ (автоинкремент) |
| team_id | BIGINT | ID команды (внешний ключ) |
| hackathon_id | VARCHAR | ID хакатона |
| hackathon_name | VARCHAR | Название хакатона |
| registered_by | VARCHAR | ID создателя, зарегистрировавшего команду |
| registered_at | TIMESTAMP | Дата регистрации |

## Связи между таблицами

- `teams.id` → `team_members.team_id` (один ко многим)
- `teams.id` → `join_requests.team_id` (один ко многим)
- `teams.id` → `team_hackathons.team_id` (один ко многим)
- `teams.creator_id` → `team_members.user_id` для связи с создателем
- `join_requests.user_id` → `team_members.user_id` после одобрения заявки

## Бизнес-логика

1. **При создании команды**:
   - Создатель автоматически добавляется как участник с флагом `is_creator = true`

2. **При подаче заявки**:
   - Проверка, не является ли пользователь уже участником
   - Проверка на наличие ожидающей заявки
   - Создание записи в `join_requests` со статусом `PENDING`

3. **При одобрении заявки**:
   - Статус заявки меняется на `APPROVED`
   - Пользователь добавляется в `team_members`
   - Создается уведомление для пользователя об одобрении

4. **При отклонении заявки**:
   - Статус заявки меняется на `REJECTED`
   - Создается уведомление для пользователя об отклонении

5. **При регистрации на хакатон**:
   - Проверка, что регистрирует создатель команды
   - Проверка на повторную регистрацию
   - Создание записи в `team_hackathons`
